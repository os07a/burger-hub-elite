# برجرهم OS — نظام إدارة المطاعم

نظام متكامل لإدارة عمليات المطعم اليومية، مبني بتقنيات حديثة وواجهة عربية بالكامل.

## الأقسام

| القسم | الوصف |
|-------|-------|
| لوحة التحكم | نظرة عامة على الأداء اليومي والإيرادات |
| حالة المشروع | تتبع تقدم المهام والمشاريع |
| مؤشر المبيعات | رسوم بيانية للمبيعات والتحليلات |
| المستشار الذكي | مستشار أعمال مدعوم بالذكاء الاصطناعي |
| سلوك المستهلك | تحليل سلوك العملاء |
| الولاء | إدارة برنامج ولاء العملاء |
| الرسائل النصية | إرسال وإدارة الرسائل |
| الطاقم | إدارة بيانات الموظفين |
| الحضور والانصراف | تسجيل ومتابعة الحضور |
| الرواتب | إدارة رواتب الموظفين |
| كاميرات المراقبة | عرض كاميرات الفروع |
| المنتجات | إدارة قائمة المنتجات والأسعار |
| المخزون | متابعة المخزون والتنبيهات |
| الموردون | إدارة الموردين والفواتير |
| تطبيقات التوصيل | ربط مع تطبيقات التوصيل |
| الأرشيف | أرشفة المستندات |
| الأرباح والنسب | تقارير الأرباح والتكاليف |

## التقنيات المستخدمة

- **Frontend:** React 18 + TypeScript + Vite 5
- **Styling:** Tailwind CSS v3 + shadcn/ui
- **Charts:** Recharts
- **Backend:** Supabase (Auth, Database, Edge Functions)
- **AI:** Google Gemini (عبر Edge Function)

## التشغيل المحلي

```bash
# 1. استنساخ المشروع
git clone <repository-url>
cd burgerhum-os

# 2. تثبيت الحزم
npm install

# 3. إعداد متغيرات البيئة
cp .env.example .env
# عدّل ملف .env بقيم مشروع Supabase الخاص بك

# 4. تشغيل المشروع
npm run dev
```

يعمل المشروع على `http://localhost:8080`

## متغيرات البيئة المطلوبة

| المتغير | الوصف |
|---------|-------|
| `VITE_SUPABASE_URL` | رابط مشروع Supabase |
| `VITE_SUPABASE_PUBLISHABLE_KEY` | مفتاح Supabase العام (anon key) |
| `VITE_SUPABASE_PROJECT_ID` | معرّف مشروع Supabase |

## النشر على Vercel

1. ارفع المشروع على GitHub
2. اربط المستودع بـ [Vercel](https://vercel.com)
3. أضف متغيرات البيئة في إعدادات المشروع على Vercel
4. اضبط الإعدادات التالية:
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
   - **Install Command:** `npm install`
5. انشر المشروع — ملف `vercel.json` يتكفل بتوجيه الصفحات تلقائياً

## النشر على Netlify

1. ارفع المشروع على GitHub
2. اربط المستودع بـ [Netlify](https://netlify.com)
3. أضف متغيرات البيئة
4. اضبط:
   - **Build Command:** `npm run build`
   - **Publish Directory:** `dist`
5. أضف ملف `public/_redirects` بالمحتوى:
   ```
   /*    /index.html   200
   ```

## ملاحظات مهمة

- **المستشار الذكي:** يستخدم `LOVABLE_API_KEY` عبر Edge Function. عند النقل خارج Lovable، استبدله بمفتاح Google Gemini API مباشر.
- **الوضع الحالي:** النظام يعمل ببيانات عرض (Demo). الربط الفعلي بقاعدة البيانات ونظام تسجيل الدخول قيد التطوير.
- **الترخيص:** جميع المكتبات مفتوحة المصدر (MIT/ISC) — آمنة للاستخدام التجاري.
