## الهدف
تنظيف صفحة تحليل المنيو، تقليل التكرار، إضافة "ربط الأقرب تلقائيًا" بنقرة واحدة، وعرض المنيو على شكل قائمة مصنّفة تظهر الشعبية والربحية بدل البطاقات الـ4 المكررة.

---

## ١) تبويب «غير مربوطة» — ربط الأقرب تلقائيًا

في `src/components/menu/UnmatchedItemsTab.tsx`:

- شريط أدوات أعلى التبويب فيه:
  - زر **«ربط الكل تلقائيًا»** — يمر على كل صنف غير مربوط، يأخذ أعلى اقتراح من `suggestProducts` بشرط `score ≥ 0.6` ويربطه دفعة واحدة (Promise.all مع تحديث متفائل + toast بإجمالي العدد المربوط والمتجاهل).
  - زر **«ربط الواضح فقط»** (افتراضي آمن) — يربط فقط ما score ≥ 0.75.
  - عدّاد: «X يمكن ربطه تلقائيًا · Y يحتاج مراجعة».
- على مستوى كل صف:
  - زر **«ربط بالأقرب»** بجانب «إنشاء منتج» — ينفّذ ربط واحد بأعلى اقتراح إن وجد (score ≥ 0.5)، ويعرض اسم المنتج المقترح ونسبة التطابق.
  - الأصناف بدون `loyverse_item_id` تُعرض رسالة "إنشاء منتج جديد فقط" بدل قائمة الاقتراحات (المنطق الحالي صحيح، نوضّحه).
- معالجة الأخطاء بشكل موحد + invalidate `menu_engineering` مرة واحدة بعد الدُفعة.

---

## ٢) عرض المنيو بدل بطاقات الأرباع الـ4

نلغي شبكة بطاقات الأرباع (Star/Plowhorse/Puzzle/Dog) المكررة لأن نفس المعلومة موجودة في «قراءة سريعة» + الجدول + المصفوفة. ونستبدلها بمكوّن جديد:

**`src/components/menu/MenuListView.tsx`** — قائمة المنيو الحقيقية مجمّعة حسب الفئة (`category`):

```
[الفئة: برغر]                              12 صنف · 8,400 ر.س هامش
  ────────────────────────────────────────
  • تشيز برغر         ●نجم      120 وحدة  ↑12%   هامش 38%
  • برغر كلاسيك       ●جواد      95 وحدة  ↓5%    هامش 22%
  • برغر مشروم        ●لغز       18 وحدة  →      هامش 61%
  • برغر بيكون        ●خاسر       3 وحدة  ↓40%   هامش 18%
[الفئة: مقبلات] ...
```

كل صف فيه:
- نقطة لون التصنيف + اسم التصنيف القصير (نجم/جواد/لغز/خاسر).
- شارة **شعبية**: مرتفعة/متوسطة/ضعيفة (مبنية على `popularity_index`).
- شارة **ربحية**: مرتفعة/متوسطة/ضعيفة (مبنية على `margin_pct` و `profitability_index`).
- اتجاه الوحدات (سهم + %).
- إيراد + هامش بالريال.

عناوين الفئات قابلة للطي، ومرتبة حسب إجمالي هامش الفئة. الأصناف بدون مبيعات تُجمّع في قسم منفصل أسفل القائمة بعنوان «بدون مبيعات في هذه الفترة (X)».

نستبدل تبويب «الجدول» بهذا العرض الجديد ونغيّر اسمه إلى **«المنيو»**.

## ٣) ترتيب الـ Layout وإزالة الحشو

ترتيب الصفحة الجديد من الأعلى للأسفل:

```text
┌─────────────────────────────────────────────────────┐
│ PageHeader (عنوان + شارة + فلتر فترة + مزامنة)    │
├─────────────────────────────────────────────────────┤
│ QuickReadStrip  (3-5 رؤى ذكية، الأهم أولاً)        │
├─────────────────────────────────────────────────────┤
│ شريط KPI مدمج  ← 4 أرقام مع مقارنة (إيراد/هامش/   │
│   متوسط الوحدات/متوسط الهامش)                       │
├─────────────────────────────────────────────────────┤
│ Tabs:  [المنيو] [المصفوفة] [غير مربوطة•] [توصيات] │
└─────────────────────────────────────────────────────┘
```

تعديلات:
- **حذف شبكة بطاقات الأرباع الـ4** (مكررة مع الجدول والمصفوفة).
- إبقاء `QuickReadStrip` كما هو (مفيد ومركّز).
- شريط الـ KPIs يبقى كما هو لكن نضيف جملة صغيرة تحته: «X نجم · Y جواد · Z لغز · W خاسر» كسطر واحد بدل 4 بطاقات.
- شريط "آخر مزامنة" يُدمج داخل badge `PageHeader` الفرعي بدل مربع منفصل.

## ٤) تفاصيل تقنية

ملفات تتغيّر/تُنشأ:

- `src/components/menu/UnmatchedItemsTab.tsx` — إضافة شريط الإجراءات الجماعية + زر «ربط بالأقرب» لكل صف، دالة `bulkAutoLink(threshold)`.
- `src/components/menu/MenuListView.tsx` *(جديد)* — قائمة منيو مجمّعة حسب الفئة مع شارات شعبية/ربحية/اتجاه وأقسام قابلة للطي.
- `src/components/menu/CategoryRow.tsx` *(جديد، اختياري)* — صف الصنف الواحد لإبقاء `MenuListView` نظيف.
- `src/pages/MenuAnalysis.tsx` — حذف بطاقات الأرباع، استبدال تبويب «الجدول» بـ `MenuListView`، تنظيف الهيدر.
- `src/hooks/useMenuEngineering.ts` — لا تغيير في المنطق، فقط نتأكد أن `popularity_index` و `profitability_index` و `category` مستخدمة في العرض الجديد (موجودة بالفعل).

منطق شارات الشعبية/الربحية:
- شعبية: `popularity_index ≥ 1.2` مرتفعة، `≥ 0.6` متوسطة، أقل ضعيفة.
- ربحية: `margin_pct ≥ 50%` مرتفعة، `≥ 30%` متوسطة، أقل ضعيفة.

«ربط الأقرب تلقائيًا» (Bulk):
```ts
async function bulkAutoLink(min = 0.75) {
  const ops = unmatched
    .map(u => ({ u, top: suggestProducts(u.display_name, products, 1)[0] }))
    .filter(x => x.u.loyverse_item_id && x.top && x.top.score >= min);
  // تنفيذ متوازي مع حدّ
  // toast: "تم ربط X صنف، تجاهل Y بسبب ضعف التطابق"
  await qc.invalidateQueries({ queryKey: ["menu_engineering"] });
}
```

## ٥) خارج النطاق

- لا تغيير على `useMenuEngineering` ولا على Edge Function `menu-engineering-advice`.
- لا تعديل قاعدة بيانات.
- المصفوفة (Scatter) تبقى كما هي — هي الأنسب لرؤية التوزيع.
