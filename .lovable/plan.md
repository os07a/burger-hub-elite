# خطة: رفع متعدد للفواتير + ملخص ذكي بعد الإضافة 📤📊

## الهدف
1. **رفع عدة صور/PDF دفعة واحدة** (حتى 10 ملفات) في تبويب "استقبال فاتورة" مع معالجة بالتوازي.
2. بعد انتهاء الدفعة، عرض **بطاقة ملخص ذكية كإشعار** تقول:
   - الفترة الزمنية المغطاة (من تاريخ → إلى تاريخ).
   - عدد الفواتير + الإجمالي بالريال.
   - **أكثر فئات الاستهلاك** (لحوم، زيوت، خضار، مشروبات...) مرتبة بالقيمة.
   - أعلى مورد + أعلى صنف من حيث المبلغ.

## التغييرات

### 1. `UploadInvoiceCard.tsx` — رفع متعدد
- تفعيل `multiple` على `<input type="file">` + قبول السحب والإفلات لعدة ملفات.
- استبدال `preview` المفرد بمصفوفة:
  ```
  files: [{ id, name, dataUri, status: pending|processing|done|error, result?, error? }]
  ```
- شبكة Thumbnails 3-4 أعمدة + أيقونة حالة لكل ملف (⏳ / 🔄 / ✅ / ❌).
- زر "تحليل الكل (N)" + شريط تقدم `(منجز/الكل)`.
- معالجة بالتوازي مع **حد 3 طلبات متزامنة** عبر `Promise.allSettled` (تجنّب استنزاف Edge Function/AI quota).
- زر حذف فردي + زر "مسح الكل".

### 2. مكوّن جديد `BatchSummaryCard.tsx`
بعد انتهاء الدفعة يظهر تلقائياً (Dialog أو بطاقة بارزة) بالمعلومات:

```text

─────────────────────────────────
📅 الفترة: 12 رمضان → 28 رمضان 1447
💰 الإجمالي: 4,235.50 ر.س
🧾 عدد الأصناف المستخرجة: 87
─────────────────────────────────
🔥 أكثر الفئات استهلاكاً:
  1. لحوم وأسماك    1,820 ر.س (43%)
  2. زيوت وسمن        612 ر.س (14%)
  3. خضار وفواكه      485 ر.س (11%)
  4. تغليف ومستهلكات  340 ر.س (8%)
─────────────────────────────────
🏪 أعلى مورد: العثيم (3 فواتير، 1,950 ر.س)
🥇 أعلى صنف: لحم بقري ميتو (520 ر.س)
─────────────────────────────────
[عرض في الأرشيف]  [دفعة جديدة]
```

### 3. منطق التجميع (Frontend بحت — بدون Backend)
- استخراج كل النتائج الناجحة من الدفعة (`results: IntakeResult[]`).
- استدعاء واحد لـ Supabase لجلب `invoices` + `invoice_line_items` للـ IDs المحفوظة.
- تجميع محلي:
  - **التاريخ من/إلى**: `min/max` على `invoice.date`.
  - **الإجمالي**: `sum(amount)`.
  - **الفئات**: ربط كل `line_item.inventory_item_id` بـ `inventory_items.category` ثم `groupBy(category) → sum(total)`.
  - **أعلى مورد**: `groupBy(supplier_name) → sum`.
  - **أعلى صنف**: `orderBy(line_item.total) limit 1`.
- البنود غير المربوطة بمخزون → فئة "غير مصنّف".

### 4. تحسين بسيط في hook
- `useBatchInvoiceSummary(invoiceIds: string[])` جديد في `useInvoiceIntake.ts` → يستخدم react-query لجلب البيانات وتجميعها.
