# ملخص الدفعة الذكي — إعادة تصميم 🧠

## المشاكل الحالية
1. الأرقام والتواريخ بصيغة عربية (٣,٩٠٨,٣٢) — المطلوب لاتيني (3,908.32).
2. البطاقة "وصفية فقط" — تعرض أرقام بدون رؤية أو تنبيه. كل البنود ظهرت كـ "غير مصنّف" والمستخدم لا يفهم لماذا ولا ماذا يفعل.
3. لا يوجد مقارنة، لا تنبيهات، لا اقتراحات.

## الهدف
بطاقة ذكية تحكي قصة وتعطي إجراءات، ليست مجرد جدول.

---

## 1. تصحيح الأرقام والتواريخ → لاتيني

استبدال:
```ts
new Intl.NumberFormat("ar-SA", ...)   // ٣,٩٠٨
toLocaleDateString("ar-SA", ...)      // ٢٦ أبريل ٢٠٢٦
```

بـ:
```ts
import { fmt, formatArabicDayMonth } from "@/lib/format";
fmt(3908.32, { maximumFractionDigits: 2 })   // → 3,908.32
formatArabicDayMonth(date)                    // → 26 أبريل (أرقام لاتينية + شهر عربي)
```

النتيجة: `26 أبريل 2026 → 1 أبريل 2026`، `3,908.32 ﷼`، `35 صنف`.

## 2. توسعة `useBatchInvoiceSummary` — حساب رؤى ذكية

في `src/hooks/useInvoiceIntake.ts` نضيف للحقول الموجودة:

- **avg_invoice**: متوسط قيمة الفاتورة في الدفعة.
- **largest_invoice**: أكبر فاتورة (مبلغ + مورد + تاريخ).
- **suppliers_count**: عدد الموردين الفريدين.
- **uncategorized_pct**: نسبة المبلغ غير المصنّف (ربط بمخزون مفقود).
- **duplicate_suspects**: فواتير من نفس المورد + نفس التاريخ + مبالغ متطابقة → احتمالية تكرار.
- **price_spikes**: مقارنة سعر كل صنف في الدفعة بمتوسط آخر 90 يوم من `invoice_line_items` لنفس `inventory_item_id`. لو ارتفع >15% → تنبيه.
- **vs_last_period**: استعلام موازي لجمع المصروفات في نفس عدد الأيام السابقة من قاعدة `invoices` (خارج هذه الدفعة) → نسبة تغير ±%.
- **top3_items**: أعلى 3 أصناف (ليس واحد) للمقارنة.

كل الحسابات تتم بـ react-query، دون تغيير قاعدة البيانات.

## 3. إعادة تصميم `BatchSummaryCard.tsx`

### A. Header ذكي مع Hero Insight
بدل "تم حفظ N فاتورة" نعرض **سطر قصة واحد** يلخص الأهم:
```
"دفعة 10 فواتير بقيمة 3,908.32 ﷼ — أعلى بنسبة 23% من نفس الفترة الشهر الماضي ⚠️"
```
لون السطر يتغير: أخضر لو أقل، أحمر لو أعلى >20%.

### B. KPI Strip (4 بطاقات صغيرة بدل 3)
```
[الفترة]  [الإجمالي]  [متوسط الفاتورة]  [عدد الموردين]
1-26 أبريل  3,908.32  390.83 ﷼/فاتورة     5 موردين
```

### C. شريط التنبيهات (جديد — الأهم) 🚨
قسم `Alerts` يعرض إشعارات قابلة للتنفيذ، كل واحد سطر مع أيقونة ولون:

| الحالة | الشكل |
|---|---|
| `uncategorized_pct > 50%` | 🟡 "82% من المبلغ غير مرتبط بمخزون — اربط الأصناف لتحليل دقيق" + زر [اربط الآن] |
| `price_spikes.length > 0` | 🔴 "ارتفع سعر «لحم بقري ميتو» 18% عن متوسط آخر 90 يوم" |
| `duplicate_suspects.length > 0` | 🟠 "احتمال فاتورة مكررة: العثيم 12 أبريل (520 ﷼) ظهرت مرتين" + زر [راجع] |
| `vs_last_period > 20%` | 🔴 "إنفاق هذه الفترة أعلى 23% من السابقة" |
| `largest_invoice > avg * 3` | 🟡 "فاتورة استثنائية: 1,820 ﷼ من العثيم — 4.6× المتوسط" |

لو ما فيه تنبيهات → سطر واحد أخضر "✅ كل شيء طبيعي، لا توجد ملاحظات".

### D. Categories — تحسين بصري
- لو فيه فئة واحدة "غير مصنّف" بنسبة 100% (الحالة الحالية في الصورة): نخفي الـ Progress ونعرض **Empty State** ذكي:
  > 📦 الأصناف لم تُربط بالمخزون بعد.
  > [ربط تلقائي بالذكاء الاصطناعي]  [ربط يدوي]
- لو في فئات متعددة: شريط أفقي مكدّس (stacked bar) واحد بدل 5 شرائط منفصلة + Legend ملون.

### E. أعلى مورد + أعلى 3 أصناف
- قائمة أعلى 3 أصناف بدل واحد (ترتيب 🥇🥈🥉).
- بطاقة المورد تضيف: "يمثل X% من إجمالي الدفعة".

### F. Footer Actions
3 أزرار:
- `[عرض في الأرشيف]` (موجود)
- `[تصدير PDF]` (جديد - مختصر للطباعة)
- `[دفعة جديدة]` (موجود)

## 4. منطق التنبيهات (frontend بحت)

دالة جديدة `buildBatchAlerts(summary): Alert[]` في `src/lib/batchAlerts.ts`:
```ts
export interface BatchAlert {
  level: "info" | "warning" | "danger" | "success";
  icon: string;
  message: string;
  action?: { label: string; onClick: () => void };
}
```

العتبات:
- `uncategorized_pct > 50` → warning
- `price_spike > 15%` → danger (لكل صنف)
- `duplicate_suspect` → warning
- `vs_last_period > +20%` → danger، بين 10-20% → warning، `< -10%` → success
- `largest_invoice / avg > 3` → info
